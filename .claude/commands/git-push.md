Commit all pending changes and push to GitHub on the current branch.

## Steps

1. **Regenerate SKILLS.md** — scan `.claude/commands/*.md` (excluding `SKILLS.md` itself) and rewrite `.claude/commands/SKILLS.md` with a fresh table. Each row is the filename (without `.md`) as the skill name and the first line of that file as the description. Use this exact format:

   ```markdown
   # Hustle Project Skills

   Custom slash commands for this project. Invoke any with `/<skill-name>` in Claude Code.

   | Skill | Description |
   |-------|-------------|
   | `/<name>` | <first line of file> |
   ...

   ---

   *Auto-regenerated by `/git-push` before every commit. To add a skill: create `.claude/commands/<name>.md` — the first line of the file becomes the description in this table.*
   ```

   Sort rows alphabetically by skill name.

2. **Check status** — show what will be committed:
   ```
   git status
   git diff --stat HEAD
   ```
   If there is nothing to commit (clean working tree and no untracked files relevant to the project), report that and stop.

2. **Review the diff** — read `git diff HEAD` for the changed files to understand what changed so the commit message is accurate. Skip binary files.

3. **Determine a commit message** — inspect `git log --oneline -5` to match the repo's commit style (`type: short description`). Types: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`. Write one line that covers the logical change; add a short body if more than one concern is touched.

4. **Stage and commit** — stage all modified tracked files plus any new project files (skip secrets like `.env`):
   ```
   git add <files>
   git commit -m "<message>\n\nCo-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>"
   ```
   Use a HEREDOC for the commit message to preserve formatting.

5. **Push** — push to the current branch on `origin`:
   ```
   git push origin <current-branch>
   ```

6. **Report** — print a short summary:
   - Commit hash and message
   - Files changed
   - Push result (branch and remote URL)
